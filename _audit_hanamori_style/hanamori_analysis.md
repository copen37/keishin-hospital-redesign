# hanamorithp.jp 分析メモ（参考パターン）

## 1) サイトマップ（トップ起点の主要導線）
- ホーム
- 病院紹介
- 外来のご案内
- 入院・面会
- 健診センター
- 医療機関の皆さまへ
- 交通アクセス
- 採用情報（診療部 / 看護部 / リハビリ）

## 2) トップのセクション構成
1. ヘッダー（電話・問い合わせ・グローバルナビ・モバイルメニュー）
2. ファーストビュー（キャッチ＋導線CTA）
3. お知らせ一覧（短いリスト + 一覧リンク）
4. 外来受付案内（電話・時間・受診導線）
5. 病院の目標/方針（番号付き情報ブロック）
6. 採用バナー
7. フッター/関連リンク

## 3) UIコンポーネント観察
- sticky寄りのヘッダー挙動（常時上部ナビ重視）
- CTAの明確化（電話・問い合わせを視認しやすく配置）
- カード/リストの情報整理（余白大きめ）
- 横並びと縦積みの切替が明確（PC→SP）
- お知らせは「日付 + タイトル」中心

## 4) 余白/タイポ/カラーの傾向
- 余白は広め、セクション間隔を十分確保
- 日本語ゴシック系フォント、見出しは太め
- クリーンな白基調 + 落ち着いたブルー/グリーン系アクセント

## 5) インタラクション
- hover（リンク・ボタン）で色変化
- スクロール時も重要CTAを見失いにくい設計
- SPはメニュー開閉で主要導線に短距離アクセス

## 6) 実装へ反映した方針（コピー回避）
- 直コピーせず、構成パターンのみ踏襲
- 独自の配色/余白/丸み/カード設計で再構成
- Hero/Bento/Ticker/Breadcrumb/固定CTAを独自実装
- 既存画像（assets/images/original）を優先利用
